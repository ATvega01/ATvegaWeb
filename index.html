Products and components

Focused building blocks for migration and data movement.

The current component set is organized around practical capabilities for bulk operations, storage access, database work, and flow control.

ATDellCASClient

Handles large/bulk operations like download, upload, delete, and related actions on Dell Centera or ECS using the CAS protocol with remarkable performance.

ATS3Client

Handles large/bulk operations like read, download, upload, delete, reports, and related actions on any S3-compatible system such as Dell ECS, NetApp StorageGrid, NetApp ONTAP, AWS S3, and similar platforms with remarkable performance.

ATDBClient

Handles large/bulk query, insert, update, and related operations on known database systems with remarkable performance.

ATGenFileClient

Handles large/bulk operations like read, download, upload, delete, reports, and related actions over general NFS and SMB protocols with remarkable performance.

ATAzureBlobClient

Handles large/bulk operations like read, download, upload, delete, reports, and related actions on Azure Blob storage.

ATflow

Master/controller that orchestrates data flows using different components for end-to-end migration workflows.

Solutions

Custom flow support for migration and sync scenarios.

We use these components to build focused data flows for both migration and continuous sync workloads, based on specific business logic and operational requirements.

Solution Pattern 1

Dell ECS CAS bucket to Azure Blob objects migration with database updates.

  • Get the meta-data and the Clip-ID from a source database.
  • Download the objects from Dell CAS storage.
  • Upload the data to Azure Blob storage.
  • Update the database with the resulting status or reference details.

Solution Pattern 2

S3-compatible source to Azure Blob migration

  • Pull content from S3-compatible systems such as Dell ECS or AWS S3.
  • Apply bulk transfer and reporting logic.
  • Publish the transferred data to Azure Blob storage.
  • Track progress and completion through the workflow controller.

Solution Pattern 3

S3-to-S3 migration.

  • Copy large data sets between S3-compatible systems.
  • Apply bulk transfer, reporting, and metadata handling.
  • Support multi-source and multi-destination migration scenarios.
  • Track transfer progress and completion through the orchestration flow.

Solution Pattern 4

Blob-to-S3 migration.

  • Move data from Azure Blob storage to S3-compatible storage.
  • Support large-scale object movement with reporting and validation.
  • Fit into broader workflows that combine object storage and database updates.
  • Reuse the same orchestration model for additional storage targets.

Solution Pattern 5

Continuous bi-directional sync between Blob & S3

  • Run scheduled or continuous sync from S3-compatible source buckets to Azure Blob containers.
  • Detect and process new or changed objects during each cycle.
  • Support reporting, retries, and integrity-oriented transfer checks.
  • Reuse ATFlow orchestration for long-running sync operations along with bulk migrations.

Solution Pattern 6

Many more migrations and sync patterns are possible.

  • Additional combinations can be assembled for file-to-object, object-to-file, and any-to-any migration needs.
  • The same toolset can run one-time migrations and ongoing sync jobs depending on the workload.
  • Each flow can be tailored to transfer volume, sync frequency, and reporting requirements.
  • The orchestration model can be reused across different storage and database environments.
